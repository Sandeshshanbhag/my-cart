## ğŸ“ Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Standalone Components** | Angular 18 best practice, no NgModules needed |
| **NgRx for state** | Predictable state management for cart, auth, search |
| **Lazy-loaded routes** | Smaller initial bundle, faster first load |
| **Firebase (BaaS)** | No custom backend needed, real-time capabilities |
| **Client-side search** | Simple for ~50 products, avoids Firestore query limits |
| **Email as doc ID** | Natural unique key for users, admins collections |
| **APP_INITIALIZER** | Auth state restored before app renders |
| **SCSS** | Nested styling, variables, mixins for maintainability |
| **Netlify hosting** | Zero-config deploys, CDN, free SSL |
